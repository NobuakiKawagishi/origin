## 第1章: Gitコマンド

### 学習ドキュメント

- Pro Git（日本語）: [Pro Git 日本語版](https://git-scm.com/book/ja/v2)

### チュートリアル

1. ターミナルでGitのバージョンを確認する（`git --version`）。
2. `git config --global user.name` と `git config --global user.email` を設定する。
3. 学習用ディレクトリで新規フォルダを作成し、`git init` を実行する。
4. `README.md` を作成して内容を1行追加する。
5. `git status` で変更状態を確認する。
6. `git add README.md` を実行してステージングする。
7. `git commit -m "docs: add initial readme"` を実行する。
8. `git log --oneline` でコミット履歴を確認する。

### 演習

- 課題1: ファイルを2回更新して、2つのコミットを作成する。
- 課題2: `git diff` と `git show` の結果を比較し、違いを説明する。
- 課題3: ブランチを作成し、変更後にmainブランチへマージする。

## 第2章: GitHub

### 事前準備

個人用のGitHubアカウントを所有している場合は、以下の手順は不要です。

1. [GitHub](https://github.com/)にアクセスして、アカウントを作成する。（個人メールアドレスでもOK）
2. GitHubにログインできることを確認する。

### 学習ドキュメント

- GitHub Docs: [Getting started with GitHub](https://docs.github.com/ja/get-started)
- GitHub Docs: [リポジトリの作成](https://docs.github.com/ja/repositories/creating-and-managing-repositories/quickstart-for-repositories)

### チュートリアル

1. GitHubにログインし、新規リポジトリを作成する。
2. ローカルリポジトリに`origin`を設定する。
3. `git push -u origin main` で初回プッシュを実行する。
4. GitHub上でREADMEが表示されることを確認する。
5. 新しいブランチを作成し、1ファイル変更してプッシュする。
6. GitHubでPull Requestを作成する。
7. 差分・説明を確認し、レビュー観点を記述する。
8. Pull Requestをマージする。

### 演習

- 課題1: 任意テーマで公開または非公開リポジトリを作成し、初回コミットを反映する。
- 課題2: ブランチ運用（featureブランチ）でPull Requestを1件完了する。
- 課題3: Issueを作成し、Issue番号をコミットメッセージに関連付ける。

## 第3章: VSCodeのGit拡張機能

### 学習ドキュメント

- VS Code公式: [Version Control in Visual Studio Code](https://code.visualstudio.com/docs/sourcecontrol/overview)
- GitLens: [GitLens Marketplace](https://marketplace.visualstudio.com/items?itemName=eamodio.gitlens)
- Git Graph: [Git Graph Marketplace](https://marketplace.visualstudio.com/items?itemName=mhutchie.git-graph)

### チュートリアル

1. `Ctrl+Shift+X`で拡張機能ビューを開く。
2. `GitLens` を検索してインストールする。
3. ソース管理ビューで変更ファイル一覧を確認する。
4. VS Code上でステージング、コミットを実施する。
5. GitLensで行ごとのコミット情報を確認する。
6. 拡張機能を無効化して再有効化し、挙動を確認する。

### 演習

- 課題1: CLI操作とVS Code操作で同じ作業を行い、操作性の違いを比較する。
- 課題2: GitLensの情報を使って、誰がどの変更を行ったかを3件調査する。
